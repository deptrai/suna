# Story 3.2: TradingView Advanced Charting

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a trader,
I want xem TradingView candlestick chart với overlay MA và RSI cho bất kỳ token nào,
so that tôi phân tích kỹ thuật trực tiếp trong Chainlens thay vì switch sang TradingView.

## Acceptance Criteria

1. **Given** user navigate đến `/dashboard/chart/[token]`
   **When** chart page load
   **Then** TradingView Lightweight Charts render candlestick OHLCV data cho token được chọn
   **And** overlay Moving Averages (MA20, MA50) hiển thị mặc định trên chart

2. **Given** user enable RSI panel
   **When** RSI overlay được toggle on
   **Then** RSI panel xuất hiện phía dưới candlestick chart với overbought (70) và oversold (30) lines
   **And** chart component dùng `"use client"` directive vì TradingView yêu cầu browser APIs — parent page vẫn là server component

## Developer Context

### Current State vs Story Impact
Hiện tại `apps/api/src/market/routes.ts` mới chỉ có các API endpoints liên quan đến protocols và general market overview (từ Story 3.1). Frontend đang hiển thị list các protocols trong `/markets`. 
Story này sẽ mở rộng tính năng phân tích kỹ thuật chuyên sâu bằng cách giới thiệu một trang riêng `chart/[token]` và ứng dụng biểu đồ chuyên nghiệp `lightweight-charts`. Nó yêu cầu fetch dữ liệu chuỗi thời gian (time-series OHLCV) thay vì dữ liệu tĩnh (static snapshots).

### Technical Requirements
- **Frontend Framework:** Next.js 15 App Router.
- **Client vs Server Components:** Page chính BẮT BUỘC là Server Component. Chart component (`trading-view-chart.tsx`) BẮT BUỘC dùng `"use client"`. Do `lightweight-charts` truy cập các API của trình duyệt (như `window`, `document`, `canvas`), cần phải import nó bằng `next/dynamic` với tùy chọn `{ ssr: false }` hoặc khởi tạo chart một cách an toàn bên trong hook `useEffect` (khi `window` đã có sẵn).
- **Thư viện Chart:** `lightweight-charts` (by TradingView). Cần kiểm tra và cài đặt (`pnpm add lightweight-charts`).
- **Styling:** Tuân thủ hệ màu "Epsilon Cyber-Glass". Sử dụng các CSS variables hoặc utility classes của TailwindCSS (Dark space black, Frosted glass, Neon accents).
- **Backend API:** Nếu mock data OHLCV, cần implement endpoint mới trong `apps/api/src/market/routes.ts` hoặc tạo mock file trực tiếp ở Frontend (khuyến nghị tạo endpoint backend để mô phỏng data fetching thật).

### Architecture Compliance
- Phải đảm bảo KHÔNG phá vỡ kiến trúc Server-first của dự án. 
- Mọi logic liên quan đến dữ liệu (như tính toán RSI, MA20, MA50) nên được thực hiện ở Server hoặc thông qua API, truyền dữ liệu sạch xuống Client. Nếu tính toán ở Client, hãy đảm bảo performance (không cản trở Main Thread).
- KHÔNG thêm thư viện charting nào khác ngoài `lightweight-charts` (hoặc tuỳ chọn cực kì tương đương) nhằm đảm bảo hiệu năng và tính crypto-native.

### Security & UX Guardrails
- Các URL chứa input của user `[token]` cần được sanitize và handle 404 / Loading state cẩn thận, tránh infinite load.
- Biểu đồ phải hoạt động mượt mà, sử dụng `ResizeObserver` để auto-resize thay vì fix cứng kích thước (pixel).
- Hiển thị Skeleton Loader trong thời gian dữ liệu OHLCV đang được load về.
