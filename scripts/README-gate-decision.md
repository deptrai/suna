# Gate Decision Enforcer Script

CI/CD script to read and enforce quality gate decisions from YAML files.

## Overview

The `enforce-gate-decision.py` script reads gate decision YAML files generated by the TEA (Test Architect) agent and enforces quality gate decisions. It's designed for CI/CD integration to automatically block or allow deployments based on gate decision criteria.

## Usage

### Basic Usage

```bash
python scripts/enforce-gate-decision.py docs/gate-decision-story-1.1.yaml
```

### Verbose Mode

```bash
python scripts/enforce-gate-decision.py docs/gate-decision-story-1.1.yaml --verbose
```

### Strict Mode

In strict mode, CONCERNS and WAIVED decisions are treated as blocking:

```bash
python scripts/enforce-gate-decision.py docs/gate-decision-story-1.1.yaml --strict
```

## Exit Codes

| Exit Code | Decision | Meaning | Action |
|-----------|----------|---------|--------|
| 0 | PASS | All criteria met | ✅ Proceed with deployment |
| 1 | FAIL | Criteria not met | ❌ Block deployment |
| 2 | CONCERNS | Minor issues | ⚠️ Proceed with caution (non-blocking) |
| 3 | WAIVED | Gate decision waived | 🔓 Non-blocking waiver |
| 4 | ERROR | Invalid file or error | ❌ Processing error |

## Gate Decision Format

The script expects a YAML file with the following structure:

```yaml
traceability_and_gate:
  gate_decision:
    decision: "PASS" | "CONCERNS" | "FAIL" | "WAIVED"
    criteria:
      p0_coverage: 100
      p1_coverage: 90
      overall_coverage: 85
      p0_pass_rate: 100
      overall_pass_rate: 95
      security_issues: 0
      critical_nfrs_fail: 0
    thresholds:
      min_p0_coverage: 100
      min_p1_coverage: 90
      min_overall_pass_rate: 90
      min_coverage: 80
```

## CI/CD Integration

### GitHub Actions

The script is integrated into the GitHub Actions workflow `.github/workflows/gate-decision-check.yml`:

- **Triggers:** Pull requests with gate decision files
- **Action:** Automatically checks all gate decision files in PR
- **Blocking:** Only FAIL decisions block merge

### Manual Trigger

You can manually trigger the gate check for a specific story:

```bash
gh workflow run gate-decision-check.yml -f story_id=1.1
```

## Decision Logic

### PASS Decision
- ✅ All P0 criteria met (100% coverage, 100% pass rate)
- ✅ All P1 criteria meet thresholds
- ✅ Overall coverage meets minimum threshold
- ✅ No security issues or critical NFR failures
- **Action:** Proceed with deployment

### CONCERNS Decision
- ✅ All P0 criteria met
- ⚠️ P1 coverage or pass rate below threshold (but not critical)
- ✅ Overall coverage meets minimum threshold
- **Action:** Proceed with deployment (non-blocking), monitor closely

### FAIL Decision
- ❌ P0 criteria not met (coverage <100% or pass rate <100%)
- ❌ Overall coverage below minimum threshold
- ❌ Security issues or critical NFR failures
- **Action:** Block deployment until issues resolved

### WAIVED Decision
- ⚠️ Decision would be FAIL based on rules
- 🔓 Business stakeholder has approved waiver
- **Action:** Proceed with deployment (non-blocking), requires approver documentation

## Validation

The script validates gate decisions against thresholds:

1. **P0 Coverage** - Must be ≥100% (blocker if not met)
2. **P0 Pass Rate** - Must be 100% (blocker if not met)
3. **P1 Coverage** - Should be ≥90% (warning if not met)
4. **Overall Coverage** - Should be ≥80% (blocker if not met)
5. **Security Issues** - Must be 0 (blocker if >0)
6. **Critical NFR Failures** - Must be 0 (blocker if >0)

## Dependencies

- Python 3.8+
- PyYAML (`pip install pyyaml`)

## Integration with Other Tools

### Pre-commit Hook

Add to `.git/hooks/pre-commit`:

```bash
#!/bin/bash
# Check gate decisions before commit
for gate_file in docs/gate-decision-story-*.yaml; do
    if [ -f "$gate_file" ]; then
        python scripts/enforce-gate-decision.py "$gate_file" --strict
        if [ $? -eq 1 ]; then
            echo "❌ Gate decision FAIL - commit blocked"
            exit 1
        fi
    fi
done
```

### Makefile Target

Add to `Makefile`:

```makefile
check-gates:
	@echo "Checking gate decisions..."
	@for gate_file in docs/gate-decision-story-*.yaml; do \
		if [ -f "$$gate_file" ]; then \
			python scripts/enforce-gate-decision.py "$$gate_file" --verbose; \
		fi \
	done
```

## Troubleshooting

### Error: Gate decision file not found

**Solution:** Ensure the gate decision YAML file exists at the specified path.

### Error: Invalid gate decision file format

**Solution:** Verify the YAML file has the correct structure with `traceability_and_gate.gate_decision` section.

### Exit Code 4: Processing error

**Solution:** Check YAML syntax and file permissions. Run with `--verbose` for detailed error messages.
