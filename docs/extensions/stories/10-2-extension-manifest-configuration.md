## Dev Notes

### Architecture Patterns and Constraints

**Manifest V3 Requirements:**
- Must use Manifest V3 (required by Chrome)
- Background pages replaced by service workers
- Content scripts run in isolated world
- Permissions must be declared explicitly
- Host permissions required for cross-origin API calls

**Extension Structure:**
- Manifest at `extension/manifest.json` (root of extension directory)
- Content script: `content-script.js` (built from TypeScript)
- Background: `background.js` (built from TypeScript)
- Side panel: `sidepanel.html` và `sidepanel.js` (built from React)
- Icons: `public/icons/` directory

**Permissions Strategy:**
- `storage`: Required for chrome.storage API (authentication tokens, settings)
- `activeTab`: Required to interact với current tab (coin detection)
- `host_permissions`: May need for API calls to backend (check CORS requirements)

**Content Script Matches:**
- Target crypto websites: CoinGecko, Binance, CoinMarketCap, etc.
- Pattern: `*://*.coingecko.com/*`, `*://*.binance.com/*`, `*://*.coinmarketcap.com/*`
- Can use wildcards for flexibility

### Project Structure Notes

**Manifest Location:**
- Manifest file at `extension/manifest.json` (root level)
- This is standard location for Chrome extensions

**Build Output:**
- Manifest references built files: `content-script.js`, `background.js`, `sidepanel.html`
- These will be generated by build process (Task 10.3)
- For now, create placeholder files hoặc reference expected build output

**Icon Structure:**
- Icons in `extension/public/icons/` directory
- Standard sizes: 16x16, 48x48, 128x128
- Format: PNG recommended
- Placeholder icons acceptable for MVP
